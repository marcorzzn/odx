ODX  Open Document eXtended Specifica Tecnica v0.1 · Bozza Marzo 2026 1. Introduzione ODX è un formato aperto per documenti digitali progettato nel 2026 per superare i limiti strutturali del PDF. Il PDF risale al 1993 e porta con sé problemi irrisolvibili: layout fisso non adattivo, OCR esterno e inaffidabile, nessun versioning nativo, modifica complessa, accessibilità opzionale. ODX eredita i pregi del PDF  portabilità, fedeltà visiva, firma digitale, archiviazione a lungo termine  eliminando ogni limite. Il risultato è un formato veloce, leggero, universale, accessibile e semanticamente ricco. 2. Architettura dei Layer Un file .odx è un container binario con segmenti separati compressi individualmente con Zstandard. Ogni segmento può essere letto, aggiornato e verificato indipendentemente dagli altri. Layer obbligatori: /meta  Metadati strutturati JSON-LD: autore, UUID, lingua, integrità crittografica. /semantic  Struttura logica XML con ruoli ARIA: titoli, paragrafi, figure, tabelle, note. /text  Testo puro UTF-8 completamente indicizzabile e ricercabile. Layer opzionali: /layout viewport. /ocr  Layer OCR nativo con confidence per parola, bounding box e candidati alternativi. /diff  Cronologia versioni stile git senza copia intera del documento. /sign  Firme digitali Ed25519 per singolo layer. 3. Struttura Binaria L'header fisso da 32 byte contiene il magic 'ODX!' seguito da versione, flags, offset della segment ODX_MAGIC = 0x4F445821 | VERSION = 0x0001 | FLAGS bitmask 4. Compressione

JPEG XL o WebP per le immagini in base al contenuto. Il font subsetting con fonttools include solo i glifi Unicode effettivamente usati nel documento. 5. Confronto con PDF Caratteristica Layout adattivo OCR nativo No Versioning No Accessibilità Opzionale Obbligatoria Compressione DEFLATE Lazy loading No Firma per layer No Open standard ODX non è un miglioramento incrementale del PDF. È una riscrittura del concetto stesso di documento digitale permanente per il 2026.
